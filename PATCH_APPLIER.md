# Patch Applier — Milestone 2 Implementation

## Overview

The **Patch Applier** is a secure, audited configuration patching system for Apache Airflow DAGs that reads repair plans generated by the LLM Repair Planner and applies constrained, validated patches to system configurations.

## Architecture

```
repair_plans.jsonl (from LLM Repair Planner)
    ↓
PatchApplier (validates, applies, commits)
    ├─ .env.inject file (environment variables)
    ├─ dags/*.py files (DAG configurations)
    └─ Git commits (every patch is committed)
    ↓
audit_log.jsonl (consumed by M3 Governance)
```

## Key Components

### 1. **PatchApplier Class**

The main class responsible for:
- Validating repair plans
- Applying patches using safe regex-only modifications
- Creating Git commits for every change
- Maintaining audit logs
- Providing batch processing capabilities

### 2. **Operator Implementations**

Five operators are supported:

#### `set_env`
- **Purpose**: Write or update environment variables in `.env.inject`
- **Parameters**: `param` (env var name), `value` (env var value)
- **Example**:
  ```json
  {
    "operator": "set_env",
    "param": "API_URL",
    "value": "https://api.example.com"
  }
  ```

#### `set_retry`
- **Purpose**: Update the `retries` value in DAG `default_args`
- **Parameters**: `dag_id` (http_dag, db_dag, file_dag), `param` (retries), `value` (number)
- **Regex Pattern**: `(["\']?retries["\']?\s*[:=]\s*)(\d+|[A-Za-z_]\w*)`
- **Example**:
  ```json
  {
    "operator": "set_retry",
    "dag_id": "http_dag",
    "param": "retries",
    "value": "5"
  }
  ```

#### `set_timeout`
- **Purpose**: Update the `execution_timeout` in DAG `default_args`
- **Parameters**: `dag_id`, `param` (execution_timeout), `value` (seconds or timedelta)
- **Regex Pattern**: `(["\']?execution_timeout["\']?\s*[:=]\s*)(timedelta\([^)]+\))`
- **Example**:
  ```json
  {
    "operator": "set_timeout",
    "dag_id": "db_dag",
    "param": "execution_timeout",
    "value": "600"
  }
  ```

#### `replace_path`
- **Purpose**: Update PATH-like environment variables
- **Parameters**: `param` (PATH variable name), `value` (new path)
- **Implementation**: Delegates to `_apply_set_env`
- **Example**:
  ```json
  {
    "operator": "replace_path",
    "param": "PYTHONPATH",
    "value": "/custom/path:/usr/bin"
  }
  ```

#### `add_precheck`
- **Purpose**: Append a precheck task to the PRECHECKS block in a DAG
- **Parameters**: `dag_id`, `value` (task code to add)
- **Markers Required**:
  ```python
  # BEGIN PRECHECKS
  ...
  # END PRECHECKS
  ```
- **Example**:
  ```json
  {
    "operator": "add_precheck",
    "dag_id": "http_dag",
    "value": "t_precheck = PythonOperator(...)"
  }
  ```

## Safety Rules (Enforced in Code)

1. **File Whitelist**: Only `.env.inject` and `dags/*.py` files can be modified
2. **No Code Execution**: Never use `eval()`, `exec()`, or `ast.parse()` on DAG files
3. **Regex-Only**: All DAG modifications use regex replacement (no AST manipulation)
4. **Human Approval Gate**: Plans with `requires_human_approval=True` are automatically rejected
5. **Git Commits**: Every patch is committed with a standard message
6. **Audit Trail**: Every action (applied or rejected) is logged to `audit_log.jsonl`


## Instance Attributes

| Attribute | Type | Purpose |
|-----------|------|---------|
| `project_root` | `Path` | Root directory of the project (Git repo) |
| `audit_log_path` | `Path` | Where audit logs are written |
| `dry_run` | `bool` | If True, simulate changes without modifying files |
| `ALLOWED_OPERATORS` | `set[str]` | Valid operators: {set_env, set_retry, set_timeout, replace_path, add_precheck} |
| `ALLOWED_TARGET_FILES` | `set[str]` | Allowlisted files: {.env.inject, dags/http_dag.py, dags/db_dag.py, dags/file_dag.py} |

## Detailed Method Documentation

### `__init__(project_root, audit_log_path, dry_run)`

Initialize the Patch Applier.

**Parameters:**
- `project_root` (str): Root directory of the project (must be a Git repository)
- `audit_log_path` (str): Path where audit log entries will be appended
- `dry_run` (bool): If True, simulate changes without modifying files

**Behavior:**
- Checks that `project_root` is a Git repository (unless dry_run)
- Creates audit_log parent directory if it doesn't exist
- Raises RuntimeError if not a Git repo (and not in dry-run mode)

---

### `apply(repair_plan) → dict`

Apply all repair actions from a single RepairPlan.

**Parameters:**
- `repair_plan` (dict): The repair plan dict with keys:
  - `plan_id` (str)
  - `failure_class` (str)
  - `repair_actions` (list[dict])
  - `requires_human_approval` (bool)

**Returns:** Dict with keys:
- `status`: "applied" | "rejected" | "partial" | "dry_run"
- `applied_actions`: list[str] of successful actions
- `failed_actions`: list[str] of failed actions
- `git_commit_hash`: str or None
- `audit_log_entry`: dict (same as audit log schema)

**Flow:**
1. Check for human approval requirement → reject if True
2. Validate all operators against ALLOWED_OPERATORS
3. Apply each repair_action by calling appropriate `_apply_*()` method
4. Collect diffs for all changed files
5. Create a single Git commit if any actions succeeded (unless dry-run)
6. Write audit log entry
7. Return result dict

---

### `apply_batch(plans_jsonl) → None`

Apply all repair plans from a JSONL file.

**Parameters:**
- `plans_jsonl` (str): Path to repair_plans.jsonl file

**Behavior:**
- Reads each line as a JSON repair plan
- Calls `apply()` for each plan
- Skips plans with `requires_human_approval=True`
- Prints summary with counts: applied, rejected, partial, pending_human_review

---

### `_apply_set_env(param, value) → str`

Write or update `param=value` in `.env.inject`.

**Regex Pattern:** None (file I/O based)

**Behavior:**
- Creates `.env.inject` if it doesn't exist
- Parses existing key=value pairs
- Updates or adds the new pair (maintains alphabetical order)
- Writes file (unless dry_run)
- Returns unified diff

---

### `_apply_set_retry(dag_id, param, value) → str`

Update retries value in DAG's `default_args`.

**Regex Pattern:**
```regex
(["\']?retries["\']?\s*[:=]\s*)(\d+|[A-Za-z_]\w*)
```

**Behavior:**
- Reads the DAG file at `dags/{dag_id}.py`
- Uses regex to find and replace the first occurrence of `retries`
- Handles both quoted and unquoted keys
- Writes file (unless dry_run)
- Returns unified diff

**Example Match:**
```
"retries": 3     →  "retries": 5
"retries" : 3    →  "retries" : 5
retries=3        →  retries=5
```

---

### `_apply_set_timeout(dag_id, param, value) → str`

Update execution_timeout in DAG's `default_args`.

**Regex Pattern:**
```regex
(["\']?execution_timeout["\']?\s*[:=]\s*)(timedelta\([^)]+\))
```

**Behavior:**
- Reads the DAG file at `dags/{dag_id}.py`
- Uses regex to find and replace the first occurrence of `execution_timeout`
- If value doesn't start with "timedelta", wraps it: `timedelta(seconds={value})`
- Writes file (unless dry_run)
- Returns unified diff

**Example Match:**
```
"execution_timeout": timedelta(seconds=300)
    →  "execution_timeout": timedelta(seconds=600)
```

---

### `_apply_replace_path(param, value) → str`

Update PATH-like environment variables.

**Implementation:** Delegates to `_apply_set_env()`

---

### `_apply_add_precheck(dag_id, value) → str`

Append a precheck task to the PRECHECKS block.

**Behavior:**
- Reads the DAG file at `dags/{dag_id}.py`
- Checks for markers:
  - `# BEGIN PRECHECKS`
  - `# END PRECHECKS`
- Inserts the `value` (task line) before `# END PRECHECKS`
- Writes file (unless dry_run)
- Returns unified diff

**Raises ValueError** if:
- DAG file doesn't exist
- PRECHECKS block markers not found

---

### `_git_commit(plan_id, failure_class) → str`

Create a Git commit for all modified files.

**Message Format:**
```
auto-repair: {plan_id} ({failure_class})
```

**Operations:**
1. `git add -u` (stage all modified files)
2. `git commit -m "auto-repair: ..."`
3. `git rev-parse --short HEAD` (get commit hash)

**Returns:** Short commit hash (7 chars)

**Raises RuntimeError** if Git operations fail

---

### `_write_audit_entry(entry) → None`

Append one JSON record to the audit log.

**Behavior:**
- Opens audit_log_path in append mode
- Writes entry as a single JSON line
- Flushes immediately

---

### `_generate_diff(filename, old_content, new_content) → str`

Generate a unified diff string.

**Returns:** Human-readable unified diff suitable for display and audit trail

---

### `_extract_target_file(diff_str, operator, param, dag_id) → str`

Extract or infer the target filename from an operation.

**Returns:** Filename (e.g., ".env.inject" or "dags/http_dag.py")

## DAG File Requirements

All DAGs must have:

1. **Imports**: Include `timedelta` from datetime
   ```python
   from datetime import datetime, timedelta
   ```

2. **Default Args**: Define with `retries` and `execution_timeout`
   ```python
   default_args = {
       "owner": "airflow",
       "retries": 3,
       "execution_timeout": timedelta(seconds=600),
   }
   ```

3. **PRECHECKS Block**: Include markers even if empty
   ```python
   with DAG(..., default_args=default_args, ...) as dag:
       # BEGIN PRECHECKS
       # Precheck tasks can be added here
       # END PRECHECKS
       
       t_task = PythonOperator(...)
   ```

## Error Handling

| Error Condition | Behavior |
|-----------------|----------|
| `requires_human_approval=True` | Plan is rejected before any modifications |
| Invalid operator | Plan is rejected; action listed in `failed_actions` |
| DAG file not found | Action fails; status becomes "partial" or "rejected" |
| Regex pattern not found | Action fails with descriptive ValueError |
| Git error | Git commit fails; status becomes "partial" |
| Invalid JSON in audit | Logged with warning; processing continues |

## Integration with M3 Governance

The audit log (`data/audit_log.jsonl`) serves as the interface to Milestone 3:

1. **M3 Safety Gate** reads the audit log
2. **Preflight Checks** validate DAG syntax after patching
3. **Human Approval CLI** displays rejected plans pending review
4. **Rollback** uses Git history to revert unsafe patches

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "Not a Git repository" | Ensure project root is a Git repo (`git init` if needed) |
| "Pattern not found" | DAG file may not have default_args or PRECHECKS block |
| Rich import error | Install: `pip install rich` |
| Git commit fails | Check that `.git/` exists and has correct permissions |
